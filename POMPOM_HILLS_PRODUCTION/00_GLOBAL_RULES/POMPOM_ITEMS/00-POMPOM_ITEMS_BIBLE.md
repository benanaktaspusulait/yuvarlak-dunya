# Pompom Items — World Signature Objects Bible

> Pompom Items are the visual identity of Pompom Hills. They appear across all episodes
> as recurring, recognizable objects. Every Pompom Item must be generated from these
> exact specs to maintain consistency.

---

## Overview

Pompom Items are soft, round, fluffy objects that belong to the Pompom Hills world.
They are NOT ordinary objects — they have a distinctive "Pompom" quality:
round shapes, fluffy textures, pastel colors, and a toy-like softness.

---

## Design Principles

1. **Always round** — no sharp edges, no angular shapes
2. **Always fluffy** — soft, cotton-like, plush texture
3. **Always pastel** — soft, warm, muted colors (never neon, never saturated)
4. **Always toy-like** — looks like a soft toy, not realistic
5. **Always friendly** — characters can touch, hold, play with them
6. **Always consistent** — same item looks the same across all episodes

## Texture Rules

- Surface must look **soft and fluffy** — like cotton or plush
- No hard surfaces, no metallic shine, no glass
- No realistic textures (no wood grain, no metal, no stone)
- Light must diffuse softly on the surface — no harsh highlights
- Shadows must be soft and warm

---

## Size Reference

All Pompom Items are **small enough to hold in two hands** — roughly 10-15cm diameter
in character scale. The exception is Pompom Cloud and Pompom Star which are environmental.

## Usage Rules

1. Every episode must include **at least 1 Pompom Item**
2. Pompom Items may be **story objects** or **background details**
3. If background, must still be visually identifiable as Pompom (round + fluffy)
4. Never use realistic alternatives (real ball, real flower, real cloud)
5. Pompom Items never break, never pop, never deflate — they are indestructible
6. Characters can hug, squeeze, bounce Pompom Items — they always return to shape

---

## OpenArt Reference Usage

When generating a shot that includes a Pompom Item:
1. Use the approved Pompom Item image as a reference
2. The item must match the reference in shape, color, and texture
3. Do not redesign the item — use the locked reference
4. If the item is not in the approved reference, do not generate it

---

## Image Generation Plan

Each Pompom Item needs:
1. **Hero image** — front view, clean background
2. **Angle views** — 3/4 view, top view
3. **In-context image** — item held by character or in environment
4. **Color variants** — if applicable (e.g., Pompom Ball in different color combos)

Generate images using the approved character/world style:
- Soft 3D cartoon style
- Rounded shapes
- Pastel colors
- Warm daylight
- Toy-like quality
