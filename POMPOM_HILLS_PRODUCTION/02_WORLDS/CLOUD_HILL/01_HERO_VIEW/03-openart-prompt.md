# Cloud Hill — OpenArt World Builder Prompt

## Purpose

This file is only the OpenArt World Builder input guide for Cloud Hill.
The approved Cloud Hill Hero View image is the primary input.
Text is secondary only.
The production bible/spec remains the deeper canon reference.
Do not paste a large structured production bible into OpenArt World Builder.

---

## World Builder Reality Check

OpenArt World Builder is a 3D spatial extrapolation tool. It may not perfectly preserve detailed shape control, cloud placement, summit readability, or the exact softness of the canon image. It may flatten the hill, distort the summit, overgrow the cloud band, add unwanted objects, or shift contrast/sharpness. Therefore, the approved Hero View image must be treated as the highest-fidelity instruction.

---

## Required Input Method

### Best Method — 1 Image Mode

- Upload the approved Cloud Hill Hero View image.
- Use the text prompt only as a supplementary style and layout hint.
- The image is the primary instruction.
- Do not add character references.
- Do not add random landscape images.
- Do not add failed generations.
- Do not add episode screenshots.
- Do not add mixed hill references from unrelated worlds.

### 2–4 Image Mode

- If OpenArt disables the text field after uploading multiple images, that is expected.
- In that case, the images become the instruction.
- Use only canon-consistent views of Cloud Hill.
- All images must match hill shape, summit shape, cloud style, sky style, lighting direction, material softness, and overall colour harmony.
- Do not mix different render styles or different hill designs.

Recommended views:

- Image 1 / Front: approved Hero View, centred iconic hill, summit and cloud band visible.
- Image 2 / Back: reverse or alternate wide view showing the dome hill from another side while preserving summit plateau and calm open landscape.
- Image 3 / Left: side angle showing the smooth dome silhouette and how the cloud band wraps near the summit.
- Image 4 / Right optional: another wide side angle, still clean and minimal, with no extra landmarks.

## OpenArt Text Prompt — Use Only In 1 Image Mode

```
Create a calm, iconic preschool world called Cloud Hill. Use the uploaded approved Hero View as the primary reference and preserve its hill shape, summit shape, cloud placement feeling, colour palette, lighting, camera feeling, and soft handcrafted Pompom Hills style.

Cloud Hill is defined by one single large smooth dome-shaped green hill that dominates the environment. The hill must feel clean, iconic, simple, and instantly recognisable. At the top is a broad flat grassy summit plateau with one small grey stone clearly visible as a summit marker.

A soft band of large fluffy white clouds sits near the upper part of the hill, wrapping gently around the summit area and creating the "Cloud Hill" identity. The clouds should support the hill silhouette, not hide it.

The environment should feel open, quiet, spacious, and contemplative. Use a wide, breathable landscape with gentle grassy foreground, a few tiny scattered white daisies, distant pastel rolling hills in the background, and a large soft blue sky with rounded fluffy clouds. Keep the sky generous and calming.

Style: Pompom Hills preschool animation world, soft pastel colours, matte handcrafted toy-like materials, rounded safe shapes, gentle storybook charm, warm morning sunlight, soft shadows, medium-low contrast, calm premium rendering.

Important identity:
- one main iconic dome hill only
- broad readable summit plateau
- one small grey summit stone
- soft fluffy cloud band near summit
- open sky
- no extra landmarks

No characters. No children. No rabbits. No animals. No birds. No houses. No buildings. No structures. No signs. No text. No roads. No paths. No fences. No bridges. No water. No trees. No bushes. No dense vegetation. No dramatic weather. No dark sky. No storm clouds. No rain. No wind effects. No HDR. No glossy plastic. No glossy CGI. No harsh shadows. No high contrast. No photorealism. No oversharpening.
```

## Identity Lock

Cloud Hill must read as:

- one iconic smooth dome hill
- broad readable summit plateau
- one small summit stone
- soft cloud band near summit
- open vast sky
- quiet grassy foreground
- distant pastel hills
- serene contemplative Pompom Hills mood

It must NOT read as:

- a generic hill landscape
- a mountain
- a flower meadow
- a forest hill
- a world with buildings or props
- a stormy cloud scene
- a path-based location
- an empty meaningless hill with no summit identity

---

## Production QA After World Build

- [ ] Is there one clearly dominant dome-shaped main hill?
- [ ] Is the summit broad, readable, and inviting?
- [ ] Is the small summit stone still visible?
- [ ] Does the cloud band support the summit instead of hiding it?
- [ ] Is the scene open and calm, with enough sky?
- [ ] Are there no buildings, roads, paths, fences, signs, or extra props?
- [ ] Are there no trees, bushes, animals, or characters?
- [ ] Is the style still soft pastel, matte, handcrafted, and preschool-safe?
- [ ] Is there no HDR, glossy CGI, oversharpening, or harsh contrast drift?
- [ ] Does it match the approved Hero View more than the text prompt?

---

## Final Usage Note

Use the World Builder output only after QA. If it flattens the hill too much, hides the summit, loses the summit stone, turns the cloud band into messy storm clouds, adds paths/trees/buildings, or becomes too glossy/HDR/contrasty, reject the world and regenerate. Do not treat a failed world as canon.
