# Pompom Hills - Environment Production Prompts

Production-ready background, location, and hero-image prompts for expanding the Pompom Hills visual world beyond character references.

Use these prompts after the base world and character references are approved in `ai-prompts/VISUAL_REFERENCES.md` and `OPENART_WORLD_PROMPT_PACK.md`.

## 1. Master Environment Style

Append this style foundation to every environment prompt unless the prompt already includes it.

```text
Pompom Hills world, soft 3D cartoon animation style, rounded shapes, pastel colours, warm daylight, gentle shadows, soft matte surfaces, toy-like appearance, toddler-safe, clean and calm composition, friendly preschool animation look, no sharp corners, no scary elements, no harsh contrast
```

## 2. Universal Negative Prompt

Use this negative prompt for all environment generations.

```text
no sharp edges, no angular buildings, no dark shadows, no realistic textures, no scary forest, no city, no cars, no weapons, no hard metal objects, no clutter, no busy background, no neon colours, no text, no watermark, no extra characters, no realistic people, no dramatic cinematic darkness
```

## 8. Recommended Generation Order

1. Generate `Pompom Hills Valley - Wide Establishing Shot`.
2. Select the warmest, softest image as the master world reference.
3. Generate `Central Village Square` and `Empty Valley Plate` using the master world reference.
4. Generate `Magical Pompom Forest`, `Oyun Evi`, and character-home locations.
5. Generate the `Round World` hero image only after the character front views and trio scale chart are approved.
6. Use the same master world reference for all weather and time variants.

## 9. Environment QA Checklist

- The background feels soft, rounded, and toddler-safe.
- The frame has room for characters and simple movement.
- There are no sharp branches, hard corners, dark tunnels, or realistic textures.
- The palette stays pastel and warm.
- The location is recognizable as Pompom Hills.
- Any character included matches the locked descriptions in `ai-prompts/VISUAL_REFERENCES.md`.
