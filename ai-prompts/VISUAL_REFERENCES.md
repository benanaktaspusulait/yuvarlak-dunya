# Pompom Hills - OpenArt Visual Reference Prompt Pack v2.0

Production prompt pack for creating consistent character, trio, world, and scene reference images for the toddler animated series **Pompom Hills**.

Use the same locked character descriptions in every generation. Do not rename characters. Do not change character colours or outfits.

## 1. Universal Base Style Prompt

**Reusable base prompt for all Pompom Hills images:**

"Pompom Hills world, soft 3D cartoon animation style, rounded shapes, pastel colours, warm daylight, gentle shadows, soft matte surfaces, toy-like appearance, toddler-safe, clean and calm composition, friendly preschool animation look"

## 2. Universal Negative Prompt

**Reusable negative prompt for all Pompom Hills images:**

"no sharp edges, no dark shadows, no realistic textures, no scary elements, no harsh lighting, no neon colours, no busy composition, no extra characters, no text, no watermark, no realistic skin, no detailed fur, no changed outfits, no angular shapes, no weapons, no cars, no city background"

## 7. OpenArt Workflow Recommendation

**Step 1: Generate world overview**
Create the Pompom Hills valley overview first. Use this as the master world mood reference for colour palette, lighting, shape language, and toy-like environmental style.

**Step 2: Generate character front views**
Generate Kiko, Mimi, and Opa front views on clean backgrounds. Use each front view as the primary character identity reference for all later images.

**Step 3: Generate side and back views**
Generate side and back views for each character using the front view as a reference image. These views should lock silhouettes, outfits, hair, ears, wings, glasses, and body proportions.

**Step 4: Generate expression sheets**
Use the approved front view for each character as the reference image. Keep outfit, colours, face shape, and proportions unchanged. Expression sheets should change only the facial expression and small body posture.

**Step 5: Generate trio comparison image**
Use the approved Kiko, Mimi, and Opa front views together as references. Generate the trio size comparison chart to lock relative scale: Kiko tallest, Mimi slightly shorter, Opa medium-sized and round.

**Step 6: Generate location references**
Use the valley overview as the world reference for every location. For character homes, also use the matching character front view as a colour and personality reference: Kiko for Kiko's house, Mimi for Mimi's burrow, Opa for Opa's tree and perch.

**Step 7: Generate first episode scenes**
For episode scenes, use three reference types together whenever possible: the relevant character front views, the trio comparison chart, and the chosen location reference. Use the universal negative prompt every time to prevent changed outfits, extra characters, busy backgrounds, and sharp or realistic details.
